## 一、OSI七层模型（必背）

### 🔍 核心要点

| 层级 | 名称    | 主要功能          | 典型协议         | 记忆口诀  |
| -- | ----- | ------------- | ------------ | ----- |
| 7  | 应用层   | 为应用程序提供服务     | HTTP、FTP、DNS | **应** |
| 6  | 表示层   | 数据格式转换、加密解密   | JPEG、MPEG    | **表** |
| 5  | 会话层   | 建立、管理、终止会话    | RPC、NetBIOS  | **会** |
| 4  | 传输层   | 端到端通信（可靠/不可靠） | TCP、UDP      | **传** |
| 3  | 网络层   | 路由选择、IP寻址     | IP、ICMP、ARP  | **网** |
| 2  | 数据链路层 | 帧传输、MAC寻址     | Ethernet、PPP | **数** |
| 1  | 物理层   | 比特传输、物理连接     | 双绞线、光纤       | **物** |

### 📝 记忆口诀

```
应表会传网数物（从上到下）
物数网传会表应（从下到上）
```

### 🔄 数据封装过程

```
应用数据 → 传输层（TCP/UDP头部）→ 网络层（IP头部）→ 
数据链路层（MAC头部+尾部）→ 物理层（比特流）
```

## 二、TCP/IP协议栈（必背）

### 🔍 核心要点

| 层级 | 名称    | 主要协议              | 对应OSI层级     | 记忆要点   |
| -- | ----- | ----------------- | ----------- | ------ |
| 4  | 应用层   | HTTP、FTP、DNS、SMTP | 应用层+表示层+会话层 | 直接面向用户 |
| 3  | 传输层   | TCP（可靠）、UDP（不可靠）  | 传输层         | 端到端通信  |
| 2  | 网络层   | IP、ICMP、ARP、RARP  | 网络层         | 路由选择   |
| 1  | 网络接口层 | Ethernet、PPP      | 数据链路层+物理层   | 物理传输   |

### 📊 TCP vs UDP对比

| 特性   | TCP       | UDP     | 记忆要点              |
| ---- | --------- | ------- | ----------------- |
| 连接   | 面向连接      | 无连接     | **TCP连，UDP断**     |
| 可靠性  | 可靠（重传、确认） | 不可靠     | **TCP可靠，UDP快**    |
| 速度   | 较慢        | 较快      | **UDP速度快**        |
| 应用   | HTTP、FTP  | DNS、视频流 | **TCP传文件，UDP传视频** |
| 首部大小 | 20-60字节   | 8字节     | **TCP头大，UDP头小**   |

## 三、IP协议（重点）

### 🔍 IPv4地址分类

| 类别 | 范围                        | 子网掩码               | 网络数     | 每个网络主机数  | 记忆要点            |
| -- | ------------------------- | ------------------ | ------- | -------- | --------------- |
| A类 | 1.0.0.0-126.255.255.255   | 255.0.0.0（/8）      | 126     | 16777214 | **1-126，/8**    |
| B类 | 128.0.0.0-191.255.255.255 | 255.255.0.0（/16）   | 16384   | 65534    | **128-191，/16** |
| C类 | 192.0.0.0-223.255.255.255 | 255.255.255.0（/24） | 2097152 | 254      | **192-223，/24** |
| D类 | 224.0.0.0-239.255.255.255 | 无                  | 无       | 无（组播）    | **224-239，组播**  |
| E类 | 240.0.0.0-255.255.255.255 | 无                  | 无       | 无（保留）    | **240+，保留**     |

### 📝 子网划分公式

```
可用主机数 = 2^(32-子网掩码位数) - 2
```

**示例**：

- 192.168.1.0/24 → 2^(32-24)-2 = 254个主机
- 192.168.1.0/26 → 2^(32-26)-2 = 62个主机

### 🔍 IPv6地址（必考）

| 地址类型   | 前缀        | 范围                                               | 特点            | 记忆要点            |
| ------ | --------- | ------------------------------------------------ | ------------- | --------------- |
| 链路本地地址 | FE80::/10 | FE80:: - FEBF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF | 不可路由，路由器接收后丢弃 | **FE80开头，不可路由** |
| 全球单播地址 | 2000::/3  | 2000:: - 3FFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF | 可路由，全球唯一      | **2000开头，可路由**  |
| 组播地址   | FF00::/8  | FF00:: - FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF | 多播，无广播        | **FF开头，组播**     |

### 📝 IPv6地址简写规则

1. **连续的0可以省略**，用`::`表示（只能出现一次）
2. **前导0可以省略**，如`0001`→`1`

**示例**：

- 完整：`2001:0db8:0000:0000:0000:ff00:0042:8329`
- 简写：`2001:db8::ff00:42:8329`

## 五、ARP协议（必考）

### 🔍 ARP工作过程

1. **查询**：主机A查询ARP缓存，是否有主机B的MAC地址
2. **广播**：如果没有，发送ARP请求（广播）：`Who has 192.168.1.2? Tell 192.168.1.1`
3. **单播**：主机B收到请求，发送ARP应答（单播）：`192.168.1.2 is at 00:11:22:33:44:55`
4. **缓存**：主机A将主机B的MAC地址存入ARP缓存（默认保留2-4小时）

### ⚠️ ARP攻击

- **特征**：网络中出现大量ARP请求（Who has）
- **危害**：ARP欺骗、ARP缓存污染
- **防御**：静态ARP绑定、ARP防护、端口安全

   - 如果匹配，发送ARP应答（单播）：
     ```
     192.168.1.2 is at 00:11:22:33:44:55
     ```
   - 如果不匹配，丢弃该请求
