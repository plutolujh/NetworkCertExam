## 1.1 什么是互联网？

### 一、硬件描述（ Nuts-and-Bolts Description）

**互联网组成部分**：
1. **端系统（End Systems）**：连接到互联网的计算机，如PC、手机、服务器
2. **通信链路（Communication Links）**：传输数据
   - 传输介质：双绞线、同轴电缆、光纤、无线电频谱
   - 传输速率：比特/秒（bps）
3. **分组交换机（Packet Switches）**：转发分组
   - 路由器（Router）
   - 链路层交换机（Link-layer Switch）
4. **ISP（Internet Service Provider）**：互联网服务提供商
   - 层级：Tier 1 → Tier 2 → Tier 3

### 二、服务描述（Services Description）

**互联网作为平台**：
- 为分布式应用提供服务的基础设施
- 应用：Web、流视频、VoIP、email、游戏等
- 应用运行在端系统上，通过套接字接口访问网络服务

### 三、协议（Protocol）**（必考）

**协议定义**：定义了两个或多个通信实体之间交换报文的格式和顺序，以及报文发送/接收或其他事件采取的行动

**人类协议 vs 网络协议**：
- 人类协议：问时间 → 回答
- 网络协议：TCP三次握手 → HTTP请求 → 响应

**互联网主要协议**：
- TCP（传输控制协议）
- IP（网际协议）
- HTTP（超文本传输协议）
- SMTP（简单邮件传输协议）

## 1.2 网络边缘（ The Network Edge）

### 一、接入网络（Access Networks）

**家庭接入**：
1. **DSL（数字用户线）**
   - 使用现有电话线
   - 下行：高达24Mbps
   - 上行：高达2.5Mbps
   - 频分复用：语音、数据分离

2. **电缆网络**
   - 使用有线电视基础设施
   - 下行：高达30Mbps
   - 上行：高达2Mbps
   - 共享广播媒体

3. **光纤到户（FTTH）**
   - 最快接入方式
   - 速率：高达Gbps

4. **拨号接入**
   - 使用调制解调器
   - 速率：56Kbps（最慢）

5. **卫星接入**
   - 用于偏远地区
   - 速率：1-2Mbps

**企业/家庭接入**：
1. **以太网（Ethernet）**
   - 局域网（LAN）
   - 速率：10Mbps-100Gbps

2. **WiFi（802.11）**
   - 无线局域网
   - 速率：最高6.9Gbps（802.11ax）

### 二、物理介质（Physical Media）

**导引介质**：
- 双绞线（Twisted Pair）：最常用，速率1-10Gbps
- 同轴电缆（Coaxial Cable）：有线电视
- 光纤（Fiber Optic）：高速、长距离

**非导引介质**：
- 无线电（Radio）：无线局域网、蜂窝网络
- 卫星：同步卫星、低轨道卫星

## 1.3 网络核心（ The Network Core）

### 一、分组交换（Packet Switching）

**分组（Packet）**：
- 应用数据被分割成分组
- 每个分组包含头部信息
- 分组通过通信链路传输

**存储转发（Store-and-Forward）**：
- 交换机必须收到整个分组后才能开始转发
- 端到端延迟 = 多个存储转发延迟之和

**排队延迟和丢失**：
- 排队等待时间
- 缓存满时分组丢失

### 二、电路交换（Circuit Switching）

**特点**：
- 专用通信线路
- 建立连接后始终可用
- 效率较低

**电路复用**：
- TDM（时分复用）
- FDM（频分复用）

**对比**：
| 特性 | 分组交换 | 电路交换 |
|------|----------|----------|
| 连接 | 无需建立 | 需要建立 |
| 资源 | 按需分配 | 专用 |
| 效率 | 高 | 低 |
| 延迟 | 可变 | 固定 |
| 适用 | 数据 | 实时语音 |

### 三、网络的网络（ A Network of Networks）

**互联网结构**：
- 顶层：Tier 1 ISP（全球ISP）
- 中间层：区域ISP
- 底层：本地ISP
- IXP（互联网交换点）：对等连接
- PoP（接入点）
- 多宿（Multi-homing）

## 1.4 延迟、丢失和吞吐量（ Delay, Loss, and Throughput）

### 一、延迟类型（Types of Delay）

**节点处理延迟（Processing Delay）**：
- 检查分组头部
- 检查错误
- 确定输出链路
- 微秒级

**排队延迟（Queuing Delay）**：
- 等待输出链路可用
- 取决于网络拥塞程度
- 可变

**传输延迟（Transmission Delay）**：
- 分组长度/带宽
- L/R
- 微秒到毫秒级

**传播延迟（Propagation Delay）**：
- 链路长度/传播速度
- 光速：3×10^8 m/s
- 毫秒级

**总延迟**：
```
d_nodal = d_proc + d_queue + d_trans + d_prop
```

### 二、排队延迟（Queuing Delay）

**流量强度（Traffic Intensity）**：
```
La = L × a / R
L：平均分组长度（bits）
a：平均分组到达速率（packets/s）
R：传输速率（bps）
```

**关键点**：
- La接近0：低排队延迟
- La接近1：高排队延迟
- La > 1：无限延迟（丢失）

### 三、吞吐量（Throughput）

**即时吞吐量**：
```
在任意时刻，接收端收到文件的速率
```

**平均吞吐量**：
```
文件大小 / 传输时间
```

**瓶颈链路**：
- 决定了整个连接的平均吞吐量
- 通常是接入网络

**示例**：
```
下载文件：
- 服务器速率：1000 Mbps
- 接入链路速率：10 Mbps
- 实际吞吐量：10 Mbps
```

## 1.5 协议层次和服务模型（ Protocol Layers）

### 一、分层架构（Layered Architecture）

**OSI七层模型**：
1. 应用层（Application）
2. 表示层（Presentation）
3. 会话层（Session）
4. 传输层（Transport）
5. 网络层（Network）
6. 数据链路层（Data Link）
7. 物理层（Physical）

**TCP/IP四层模型**：
1. 应用层（Application）
2. 传输层（Transport）
3. 网络层（Network）
4. 网络接口层（Link）

### 二、封装（Encapsulation）

**封装过程**：
```
应用层数据
    ↓ 添加传输层头部 → 传输层 segment
    ↓ 添加网络层头部 → 网络层 datagram
    ↓ 添加链路层头部和尾部 → 链路层 frame
    ↓ 物理层传输 → 比特流
```

**解封装过程**：
```
比特流
    ↓ 链路层去除头部尾部 → 网络层 datagram
    ↓ 网络层去除头部 → 传输层 segment
    ↓ 传输层去除头部 → 应用层数据
```

### 三、服务模型

**面向连接服务**：
- TCP
- 提供可靠传输、流量控制、拥塞控制

**无连接服务**：
- UDP
- 提供不可靠传输

## 1.6 网络攻击（ Networks Under Attack）

### 一、恶意软件（Malware）

**类型**：
- 病毒（Virus）：需要用户交互
- 蠕虫（Worm）：自我复制
- 间谍软件（Spyware）
- 勒索软件（Ransomware）

### 二、攻击方式

**拒绝服务（DoS）**：
- 弱点攻击
- 带宽 flooding
- 连接 flooding

**分布式拒绝服务（DDoS）**：
- 多个源攻击

**packet sniffing**：
- 嗅探网络数据包
- Wireshark工具

**IP欺骗**：
- 伪装源IP地址

**中间人攻击（MITM）**：
- 拦截通信

## 核心记忆要点（必背）

### 📝 关键概念
1. **互联网组成**：端系统+通信链路+分组交换机+ISP
2. **协议**：定义报文交换格式和顺序
3. **接入方式**：DSL、电缆、光纤、WiFi

8. **OSI七层**：应用-表示-会话-传输-网络-数据链路-物理
9. **TCP/IP四层**：应用-传输-网络-网络接口
10. **封装过程**：应用→传输→网络→链路→物理

