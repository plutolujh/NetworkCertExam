## 一、防火墙技术

**防火墙部署位置与作用：**

| 位置 | 作用 |
|------|------|
| 互联网边界（防火墙1） | 边界防护，阻挡外部攻击，控制外部对DMZ访问 |
| DMZ与内网之间（防火墙2） | 分区隔离，保护内网，防止DMZ被突破后蔓延 |

**防火墙核心功能：**
- 访问控制（基于IP、端口、协议）
- 状态检测（跟踪连接状态）
- NAT地址转换
- 日志记录与告警

**防火墙vs IDS：**
- 防火墙是**主动防御**（主动阻止）
- IDS是**被动检测**（检测告警，需联动才能阻断）

## 二、IDS入侵检测系统

**部署方式：**
- **旁路部署**（Tap/SPAN镜像口）
- 与主链路并行，不影响网络性能

**IDS能力：**
- 特征库匹配检测已知攻击
- 行为分析检测异常流量
- 关联分析发现APT攻击

**IDS位置优缺点：**
- 优点：不影响性能、可双向检测
- 缺点：无法主动阻断、需联动防火墙

## 三、DMZ非军事区

**为什么需要DMZ：**
- 放置对外提供服务的服务器（Web、Mail、DNS等）
- 这些服务器被攻击风险较高，与内网隔离
- 即使DMZ被攻破，内网仍有防火墙2保护

**拓扑示例：**
```
[互联网] ---防火墙1--- [DMZ区] ---防火墙2--- [核心交换机] --- [内网]
```

## 四、网络安全态势感知平台

**核心功能（5个要点）：**

1. **全网安全事件采集** — 汇总防火墙、IDS、日志、流量等数据
2. **关联分析** — 多源数据关联，发现攻击链
3. **威胁检测** — 特征库+行为分析，检测APT、零日漏洞
4. **可视化展示** — 网络拓扑、资产状态、安全态势一目了然
5. **应急响应联动** — 与防火墙、IDS联动，自动阻断

**在防护体系中的定位：**
- **大脑角色**：分散设备串联成整体防护
- **检测未知威胁**：弥补防火墙、IDS对未知攻击的不足
- **统一指挥调度**：协调各安全设备协同响应

## 五、ACL访问控制列表

**工作原理：**
1. 按规则顺序**逐条匹配**
2. **匹配即执行**（permit或deny）
3. **未匹配隐含deny**
4. 匹配基于：协议、源地址、源端口、目的地址、目的端口

**配置示例：**
```
# 内网段：10.1.0.0/16
# 服务器区段：10.2.0.0/16

rule permit tcp 10.1.0.0 0.0.255.255 10.2.0.0 0.0.255.255 eq 80
rule permit tcp 10.1.0.0 0.0.255.255 10.2.0.0 0.0.255.255 eq 443
rule permit tcp 10.1.0.0 0.0.255.255 10.2.0.0 0.0.255.255 eq 3389
rule deny ip any any
```

## 六、VPN技术对比

**VPN三大应用场景：**
1. 远程移动办公
2. 总分支机构Site-to-Site互联
3. 第三方人员安全接入

**SSL VPN vs IPSec VPN：**

| 对比项 | SSL VPN | IPSec VPN |
|--------|---------|-----------|
| 典型场景 | 移动办公、Web应用远程接入 | 固定分支机构间Site-to-Site |
| 客户端要求 | 仅需浏览器 | 专业客户端或网关 |
| NAT穿越 | 容易 | 困难（需配置NAT-T） |
| 安全性 | 高（SSL/TLS） | 高（隧道加密） |
| 适用用户 | 出差人员、远程办公 | 固定办公点之间 |

**记忆口诀：移动办公用SSL，站点互联用IPSec**
