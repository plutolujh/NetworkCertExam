# 2024年11月网络规划设计师 下午案例分析真题

## 第一题：网络系统规划设计与优化（约15分）

### 背景描述

某高校智慧校园网络需要升级改造，当前校园网采用传统的三层架构（核心-汇聚-接入），随着智慧校园业务发展，存在以下问题：
1. 现有网络难以支撑大量物联网终端的接入
2. 网络配置复杂度高，运维效率低
3. 无线网络覆盖不足，存在信号盲区
4. 出口带宽利用率不均衡

### 问题

**【问题1】**（4分）
针对高校智慧校园网络的现状，在不改变现有三层架构的基础上，有哪些网络优化措施可以解决上述问题？请给出至少4种优化措施。

**【问题2】**（5分）
随着SDN技术的成熟，计划在校园网中引入SDN控制器。请说明SDN控制器的部署位置，并分析引入SDN后对校园网络的改进作用（至少4点）。

**【问题3】**（4分）
在无线网络补盲项目中，计划在图书馆部署一套高密放装无线网络。请说明在部署无线网络时需要考虑的关键因素（至少4个方面）。

**【问题4】**（2分）
校园网现有两条出口链路：教育网100Mbps和运营商200Mbps。请设计一种出口负载均衡策略，使得内网用户访问学术资源时优先使用教育网出口。

## 第二题：网络安全防护方案设计（约15分）

### 背景描述

某企业网络拓扑如图所示，计划建设完善的网络安全防护体系。

```
[互联网] ---防火墙1--- [DMZ区] ---防火墙2--- [核心交换机] --- [内网]
                |                                    |
              [IDS]                                [服务器区]
                |                                    |
              [VPN网关]---------------------------|
```

### 问题

**【问题1】**（4分）
根据网络安全防护体系架构图，说明防火墙1、防火墙2、IDS和VPN网关各自的安全防护作用，并指出IDS部署位置的优缺点。

**【问题2】**（5分）
企业计划部署一套综合网络安全态势感知平台。请说明该平台的核心功能，以及在网络安全防护体系中发挥的作用。

**【问题3】**（4分）
在网络安全策略配置中，计划在防火墙2上部署ACL策略。请说明ACL的基本工作原理，并给出内网访问服务器区的ACL规则示例（要求包含协议、源地址、目的地址、动作等要素）。

**【问题4】**（2分）
VPN技术在企业网络中主要用于哪些场景？并说明SSL VPN和IPSec VPN各自的适用场景。

## 第三题：传输网络规划与设计（约15分）

### 背景描述

某市级政务网络需要建设覆盖全市的骨干传输网，要求提供大带宽、高可靠性、可扩展的传输服务。

### 问题

**【问题1】**（4分）
目前主流的传输网络技术包括SDH、MSTP、OTN和DWDM。请比较这四种技术的特点（从带宽能力、业务适配性、智能性等维度），并说明OTN技术的主要优势。

**【问题2】**（5分）
在骨干传输网设计中，计划采用环形组网方式以提供链路保护。请说明环形保护的两种主要方式（通道保护环和复用段保护环）的区别，并分析各自的适用场景。

**【问题3】**（4分）
传输网的可靠性设计需要考虑哪些因素？请从设备冗余、链路冗余、网络拓扑等方面进行说明。

**【问题4】**（2分）
如果该政务网络需要支持SDH业务、以太网业务和视频业务等多种业务类型，从技术成熟度和兼容性角度考虑，推荐采用哪种传输技术方案？请说明理由。

## 第四题：网络存储与计算平台设计（约15分）

### 背景描述

某企业数据中心需要建设云存储和计算平台，用于支撑企业业务系统的数据存储和处理需求。

### 问题

**【问题1】**（4分）
传统的DAS、NAS和SAN存储架构各有特点。请比较这三种存储架构的连接方式、扩展性、适用场景等方面的差异。

**【问题2】**（5分）
随着数据量增长，企业计划建设分布式存储系统。请说明分布式存储的核心技术架构（至少4个组件），以及分布式存储相对于传统存储的优势。

**【问题3】**（4分）
在云计算平台建设中，计划采用计算密集型云主机和存储密集型云主机混合部署的方案。请说明两种云主机在配置上的主要差异，并设计一种合理的资源调度策略。

**【问题4】**（2分）
数据备份是数据保护的重要手段。请说明全量备份、增量备份和差异备份三种备份策略的特点，并分析各自的优缺点。

## 第五题：网络故障诊断与排除（约15分）

### 背景描述

某公司网络拓扑如图所示，最近出现以下故障现象：
- 故障1：研发部部分员工反映访问文件服务器速度很慢，但Ping服务器延迟正常
- 故障2：交换机SW3下联用户反映偶尔出现掉线情况

```
[路由器] --- [核心交换机SW1] --- [汇聚交换机SW2] --- [接入交换机SW3] --- [用户区]
                |                                           |
                |--- [服务器区]                            [文件服务器]
```

### 问题

**【问题1】**（4分）
针对故障1，研发部员工Ping服务器延迟正常，但访问速度很慢。请分析可能的原因，并说明需要使用哪些诊断工具进行排查。

**【问题2】**（5分）
针对故障2，交换机SW3下联用户偶尔掉线。请从网络环路、端口速率协商、链路干扰等方面分析可能的原因，并给出相应的解决方法。

**【问题3】**（4分）
在故障排查过程中，使用Sniffer抓包发现大量TCP重传现象。请分析造成TCP重传的常见原因，并说明如何定位具体的故障点。

**【问题4】**（2分）
网络故障的预防措施包括哪些方面？请从网络监控、配置管理、变更管理等方面进行说明。

### 第一题答案要点

**【问题1】优化措施：**
1. 部署SDN控制器，实现网络集中管控
2. 增加无线AP数量，消除信号盲区
3. 部署多出口负载均衡，优化带宽利用
4. 简化网络配置，标准化VLAN规划
5. 部署网络自动化运维工具

**【问题2】SDN控制器：**
- 部署位置：核心层或独立集中部署
- 改进作用：
  - 网络可编程，配置自动化
  - 流量调度灵活，优化带宽利用
  - 故障快速响应，自动化恢复
  - 统一管控，简化运维

**【问题3】无线网络部署因素：**
1. 覆盖范围和信号强度
2. 接入容量和并发用户数
3. 信道规划和干扰规避
4. 终端兼容性
5. 安全策略（Portal/802.1X）

**【问题4】出口负载均衡策略：**
- 基于目的地址的策略路由
- 教育网地址段走教育网出口
- 其他流量默认走运营商出口

### 第二题答案要点

**【问题1】安全设备作用：**
- 防火墙1：边界防护，互联网访问控制
- 防火墙2：分区隔离，内网安全防护
- IDS：入侵检测，流量分析预警
- VPN网关：远程接入安全加密

IDS旁路部署优点：不影响网络性能，可双向检测
缺点：无法阻止攻击，需要联动防火墙

**【问题2】态势感知平台功能：**
1. 全网安全事件采集与分析
2. 威胁检测与预警
3. 资产可视化展示
4. 应急响应联动
5. 态势预测与趋势分析

**【问题3】ACL工作原理：**
按规则顺序匹配，完成过滤
示例：
```
permit tcp 10.1.0.0 0.0.255.255 10.2.0.0 0.0.255.255 eq 80
deny ip any any
```

**【问题4】VPN场景：**
- SSL VPN：移动办公、Web应用接入
- IPSec VPN： site-to-site互联

### 第三题答案要点

**【问题1】传输技术比较：**
| 技术 | 带宽能力 | 业务适配 | 智能性 |
|------|---------|---------|--------|
| SDH | 低 | 语音/电路 | 较弱 |
| MSTP | 中 | 混合业务 | 一般 |
| OTN | 高 | 全业务 | 较强 |
| DWDM | 极高 | 波长业务 | 弱 |

OTN优势：超大带宽、标准帧结构、前后兼容

**【问题2】环形保护：**
- 通道保护：1+1备份，倒换快(50ms)
- 复用段保护：共享备份，容量效率高

**【问题3】可靠性设计因素：**
1. 设备关键板卡冗余
2. 链路双归属
3. 环形/网状拓扑
4. 电源冗余

**【问题4】推荐方案：**
MSTP或OTN，技术成熟，多业务兼容

### 第四题答案要点

**【问题1】存储架构比较：**
| 架构 | 连接方式 | 扩展性 | 适用场景 |
|------|---------|--------|---------|
| DAS | 直连 | 差 | 本地存储 |
| NAS | 网络 | 中 | 文件共享 |
| SAN | 专用网络 | 好 | 数据库 |

**【问题2】分布式存储架构：**
- 元数据管理节点
- 存储节点
- 客户端/网关
- 管理平面

优势：线性扩展、高可用、低成本

**【问题3】云主机配置差异：**
- 计算型：CPU核心多，主频高
- 存储型：磁盘容量大，IO高

调度策略：负载均衡+业务优先级

**【问题4】备份策略：**
- 全量：完整备份，恢复简单，占用空间大
- 增量：只备份变化，恢复复杂
- 差异：备份基准+变化，平衡方案

### 第五题答案要点

**【问题1】故障1分析：**
可能原因：服务器带宽瓶颈、端口速率协商异常、网络攻击导致拥塞
排查工具：Wireshark、流量分析器、带宽测试工具

**【问题2】故障2分析：**
可能原因：网线质量差、水晶头接触不良、端口故障、广播风暴
解决方法：更换网线、检修端口、启用STP

**【问题3】TCP重传原因：**
- 网络丢包
- 链路拥塞
- 设备性能瓶颈
- 路由不稳定

定位方法：分段抓包测试、对比各段丢包率

**【问题4】预防措施：**
1. 部署网络监控系统
2. 定期巡检配置备份
3. 变更审批流程
4. 应急预案演练
