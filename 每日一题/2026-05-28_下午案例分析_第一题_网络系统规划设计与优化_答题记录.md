## 问题2：SDN控制器部署（5分）

### 学生答案
SDN就是软件定义网络，传统的网络管理是每台机器都有自己的控制规则，SDN则整个网络控制规则在一处控制，统一配置统一下发。控制应该放在核心。减少配置工作量，更方便管理优化升级。

### 参考答案
**部署位置**：核心层，或独立集中部署

**改进作用（至少4点）**：
1. 网络可编程，配置自动化
2. 流量调度灵活，优化带宽利用
3. 故障快速响应，自动化恢复
4. 统一管控，简化运维
5. 支持新业务快速部署

### 问题分析
| 学生回答 | 问题 |
|---------|------|
| SDN概念正确 | ✅ 正确，但不需要在答题时解释定义 |
| "放在核心" | 不够准确 — 应该是"核心层**或**独立集中部署" |
| "减少配置工作量" | 方向对，但太笼统 |
| "方便管理优化升级" | 缺少分点展开，没有明确说出4个具体作用 |

### 得分：约2.5~3分 / 5分

### 薄弱知识点
1. **SDN控制器部署位置** — 需要掌握"核心层或独立部署"两种方式
2. **SDN改进作用** — 需要分点记忆4个明确作用：可编程、流量调度、故障恢复、统一管控
3. **SDN的北向/南向接口** — 了解OpenFlow等南向协议

## 问题4：出口负载均衡策略（2分）

### 学生答案
可以通过域名识别和IP地址段识别来区分流量访问的是否是学术类网址，如果是走教育网，否则看负载，用户的分配正常是1:2分配，同时考虑流量负载情况。

### 参考答案
基于目的地址的策略路由（PBR）：
- 教育网地址段走教育网出口
- 其他流量默认走运营商出口
- 可结合DNS分流

### 问题分析
| 学生回答 | 问题 |
|---------|------|
| 域名识别 + IP地址段识别 | ✅ 完全正确，命中核心方法 |
| 学术类走教育网 | ✅ 正确 |
| 1:2分配 | ✅ 合理 — 100Mbps:200Mbps本身就是1:2 |
| 考虑流量负载 | ✅ 体现负载均衡意识 |

### 得分：约1.5~1.8分 / 2分

### 薄弱知识点
1. **策略路由（PBR）** — 需要掌握配置命令和原理
2. **DNS分流** — 了解如何通过DNS解析结果决定路由
3. **NAT与策略路由的区别** — 需要区分

## 需要强化的知识点清单

### 🔴 高优先（必须掌握）

1. **SDN控制器**
   - 部署位置：核心层或独立部署
   - 改进作用：可编程、流量调度、故障恢复、统一管控
   - 关联协议：OpenFlow

2. **无线网络部署**
   - 信道规划（2.4GHz 1/6/11信道；5GHz可用信道更多）
   - 同频干扰规避
   - AP密度计算（高密场景每AP覆盖人数）

3. **多出口负载均衡**
   - 策略路由（PBR）原理和配置
   - DNS分流方法
   - 教育网地址段识别

4. **网络架构优化思路**
   - 物联网接入问题 → SDN/虚拟化
   - 运维复杂 → 自动化工具/标准化
   - 无线盲区 → AP补点
   - 带宽不均 → 多出口负载均衡

### 🟡 中等优先

1. **VLAN规划** — 不仅仅是为了减小冲突域，更重要的是网络分段和安全隔离
2. **链路聚合** — Eth-Trunk配置和LACP协议
3. **无线安全策略** — Portal认证 vs 802.1X
