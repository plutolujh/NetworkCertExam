## 1. SDN应用场景和作用（🔴 高优先）

### 核心概念

SDN（Software Defined Networking，软件定义网络）将网络控制平面与数据转发平面分离，由集中式控制器统一管理。

### SDN三大核心价值

| 价值 | 说明 |
|-----|------|
| **网络可编程** | 通过API开放网络能力，支持自动化配置 |
| **集中管控** | 控制器全局视图，统一调度全网资源 |
| **灵活扩展** | 新业务部署无需修改硬件，软件定义策略 |

### SDN改进作用（必须熟记4点）

1. **网络可编程，配置自动化** — 传统网络每台设备单独配置，SDN通过控制器统一下发，配置工作量大幅减少
2. **流量调度灵活，优化带宽利用** — 控制器全局视角，基于应用需求动态调整流量路径
3. **故障快速响应，自动化恢复** — 自动检测故障，自动调整路由，无需人工干预
4. **统一管控，简化运维** — 所有网络设备由控制器集中管理，运维效率提升

### SDN控制器部署位置

| 方式 | 说明 |
|-----|------|
| 核心层部署 | 与核心交换机合设，适合中小型网络 |
| 独立集中部署 | 独立服务器，适合大型网络（推荐） |

### 相关协议

- **OpenFlow** — 主流南向接口协议
- **NETCONF** — 网络配置协议，XML编码

## 2. 无线信道规划（🔴 高优先）

### 高密场景无线网络部署关键因素

**官方答案的5个关键因素**：
1. **覆盖范围和信号强度**
2. **接入容量和并发用户数**
3. **信道规划和干扰规避** ← 高密场景最关键
4. **终端兼容性**
5. **安全策略**（Portal/802.1X）

### 信道规划核心知识

**2.4GHz频段信道**（中国可用）：
- 1、6、11 — 三个互不干扰信道
- 相邻信道会产生干扰

**5GHz频段信道**：
- 可用信道更多：149、153、157、161、165等
- 干扰更小，容量更大

### AP密度计算

高密场景（如图书馆）每AP覆盖人数通常按20-30人设计

### 相关技术

- **AC+AP架构** — AC无线控制器集中管理AP
- **无缝漫游** — 用户移动时AP切换，零感知
- **负载均衡** — 多AP间用户均匀分布

## 3. 多出口负载均衡（🔴 高优先）

### 问题场景

某高校有两条出口链路：
- 教育网100Mbps
- 运营商200Mbps

需求：访问学术资源优先走教育网

### 解决方案：策略路由（PBR）

**策略路由（Policy-Based Routing，PBR）**：
- 基于目的地址选择路由路径
- 基于源地址选择路由路径
- 基于应用/端口选择路由路径

### 配置思路

```
1. 定义学术资源地址段（教育网IP段）
2. 配置策略路由：
   - 目的地址 = 学术资源 → 走教育网出口
   - 其他流量 → 默认走运营商出口
3. 可以结合DNS分流
```

### 多出口负载均衡策略

| 策略 | 说明 |
|-----|------|
| **基于目的地址** | 学术资源走教育网，其他走运营商 |
| **基于源地址** | 某些内网区域指定出口 |
| **基于流量负载** | 根据带宽比例1:2分配流量 |
| **备份策略** | 主出口故障自动切换到备用出口 |

### 相关技术

- **NAT** — 网络地址转换
- **DNS分流** — 不同DNS解析结果返回不同IP
- **健康检查** — 检测出口链路故障自动切换

## 4. 网络架构优化 vs 运维管理

### 核心区别

| 维度 | 网络架构优化 | 网络运维管理 |
|-----|------------|-------------|
| **层面** | 网络设计/规划层面 | 网络日常运营层面 |
| **目标** | 解决网络本身的能力问题 | 保证网络正常运行 |
| **手段** | 设备升级、架构调整 | 配置管理、监控、排障 |
| **回答角度** | "网络需要怎么改造" | "网络怎么维护" |

### 问题1的4个现状 → 对应架构优化

| 现状 | 问题本质 | 架构优化方向 |
|-----|---------|------------|
| 物联网终端接入困难 | 网络扩展性不足 | SDN集中管控、自动化部署 |
| 配置复杂度高 | 网络标准化不足 | 标准化VLAN、自动化运维 |
| 无线覆盖不足 | 无线覆盖范围不够 | 增加AP、补盲 |
| 出口带宽不均 | 多出口负载不均衡 | 多出口负载均衡、策略路由 |

### 常见混淆点

| 运维管理措施（错） | 架构优化措施（对） |
|------------------|------------------|
| 查杀病毒 | 部署IDS/防火墙 |
| 流量限速 | SDN流量调度 |
| 用户权限管理 | 网络分段隔离 |

### 记忆口诀
> "架构优化看网络本身，运维管理看网络运行"
> "案例分析考设计，不是考网工"

## 5. 策略路由(PBR) 知识点补充

### 基本原理

**传统路由**：根据目的地址查路由表，选择下一跳

**策略路由**：根据多种条件（源地址、目的地址、端口、应用等）综合决策

### ACL与策略路由的关系

```
ACL用于匹配条件 → 策略路由根据匹配结果执行动作
```

### 标准ACL配置示例

```
# 允许内网访问服务器区的HTTP/HTTPS
rule permit tcp 10.1.0.0 0.0.255.255 10.2.0.0 0.0.255.255 eq 80
rule permit tcp 10.1.0.0 0.0.255.255 10.2.0.0 0.0.255.255 eq 443
rule deny ip any any
```

### PBR配置要点

1. **定义ACL** — 匹配需要特殊处理的流量
2. **定义路由策略** — 指定流量走哪个出口
3. **接口应用** — 在相应接口调用策略
