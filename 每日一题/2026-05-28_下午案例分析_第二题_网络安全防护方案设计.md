---

## 题目回顾

某企业网络拓扑如图所示，计划建设完善的网络安全防护体系。

```
[互联网] ---防火墙1--- [DMZ区] ---防火墙2--- [核心交换机] --- [内网]
                |                                    |
              [IDS]                                [服务器区]
                |                                    |
              [VPN网关]---------------------------|
```

### 问题

**【问题1】**（4分）
根据网络安全防护体系架构图，说明防火墙1、防火墙2、IDS和VPN网关各自的安全防护作用，并指出IDS部署位置的优缺点。

**【问题2】**（5分）
企业计划部署一套综合网络安全态势感知平台。请说明该平台的核心功能，以及在网络安全防护体系中发挥的作用。

**【问题3】**（4分）
在网络安全策略配置中，计划在防火墙2上部署ACL策略。请说明ACL的基本工作原理，并给出内网访问服务器区的ACL规则示例（要求包含协议、源地址、目的地址、动作等要素）。

**【问题4】**（2分）
VPN技术在企业网络中主要用于哪些场景？并说明SSL VPN和IPSec VPN各自的适用场景。

## 解题思路框架

### 问题1：安全设备作用分析（4分）

**拓扑结构解析**：
```
互联网 → 防火墙1 → DMZ区 → 防火墙2 → 核心交换机 → 内网
              ↓                        ↓
             IDS                   服务器区
              ↓
          VPN网关（绕过防火墙2连接核心交换机）
```

| 设备 | 位置 | 作用 |
|------|------|------|
| **防火墙1** | 互联网与DMZ之间 | 边界防护，控制互联网对DMZ的访问，阻挡外部攻击 |
| **防火墙2** | DMZ与核心交换机之间 | 分区隔离，保护内网，防止DMZ区被攻击后蔓延到内网 |
| **IDS** | 旁路部署在防火墙1和DMZ之间 | 入侵检测，流量分析预警，检测攻击行为 |
| **VPN网关** | 旁路连接互联网和核心交换机 | 远程接入安全加密，移动办公用户接入 |

**IDS部署位置优缺点**：
- **旁路部署优点**：不影响网络性能，可双向检测攻击流量
- **旁路部署缺点**：无法主动阻止攻击，只能检测和告警，需要联动防火墙才能阻断

### 问题2：网络安全态势感知平台（5分）

**核心功能**：
1. **全网安全事件采集与分析** — 日志、流量、告警集中采集和关联分析
2. **威胁检测与预警** — 基于特征库和行为分析检测APT攻击、零日漏洞
3. **资产可视化展示** — 网络拓扑、资产状态、安全状态可视化
4. **应急响应联动** — 与防火墙、IDS等设备联动，自动阻断攻击
5. **态势预测与趋势分析** — 基于历史数据预测安全趋势

**在防护体系中的作用**：
- **大脑作用** — 将分散的安全设备串联起来，形成整体防护
- **检测作用** — 发现未知威胁和高级攻击
- **指挥作用** — 统一调度各安全设备协同防御

### 问题3：ACL原理与配置（4分）

**ACL基本工作原理**：
1. 按规则顺序匹配（深度优先匹配）
2. 匹配即执行动作（permit或deny）
3. 未匹配则隐含deny
4. 逐包过滤，逐流检测

**内网访问服务器区的ACL规则示例**：

```
# 假设内网段：10.1.0.0/16
# 服务器区段：10.2.0.0/16
# 允许HTTP、HTTPS访问

rule permit tcp 10.1.0.0 0.0.255.255 10.2.0.0 0.0.255.255 eq 80
rule permit tcp 10.1.0.0 0.0.255.255 10.2.0.0 0.0.255.255 eq 443
rule permit tcp 10.1.0.0 0.0.255.255 10.2.0.0 0.0.255.255 eq 3389  # 远程桌面
rule deny ip any any
```

**或者基于方向的标准ACL**：
```
access-list 100 permit tcp 10.1.0.0 0.0.255.255 10.2.0.0 0.0.255.255 eq 80
access-list 100 permit tcp 10.1.0.0 0.0.255.255 10.2.0.0 0.0.255.255 eq 443
access-list 100 deny tcp any any
```

### 问题4：VPN应用场景（2分）

**VPN主要应用场景**：
1. **远程移动办公** — 员工在外地通过互联网安全接入企业内网
2. **Site-to-Site互联** — 总分支机构之间建立加密隧道
3. **外包人员/合作伙伴接入** — 第三方人员安全访问内网资源

**SSL VPN vs IPSec VPN对比**：

| 特性 | SSL VPN | IPSec VPN |
|------|---------|-----------|
| 典型场景 | 移动办公、Web应用接入 | Site-to-Site互联 |
| 部署方式 | 仅需浏览器，无需安装客户端 | 需要专业客户端或网关设备 |
| 适用用户 | 出差人员、远程办公 | 固定分支机构间互联 |
| 安全性 | 较高（使用SSL/TLS） | 高（隧道加密） |
| 穿越NAT | 容易 | 困难（需要NAT穿透配置） |

**标准答案**：
- **SSL VPN**：适合移动办公人员、远程访问内网Web应用、B/S架构系统
- **IPSec VPN**：适合固定场所之间的Site-to-Site互联，要求高安全性

## 知识点关联

| 知识点 | 关联内容 |
|-------|---------|
| 防火墙 | 分区隔离、访问控制、状态检测 |
| IDS/IPS | 入侵检测、入侵防御、特征库、行为分析 |
| DMZ | 非军事区、对外服务区、隔离区 |
| 态势感知 | 大数据分析、威胁情报、关联分析 |
| ACL | 包过滤、状态检测、策略匹配 |
| VPN | SSL/TLS、IPSec、隧道模式、传输模式 |
| 零信任 | 持续验证、最小权限原则 |

## 答题技巧总结

1. **画拓扑帮助理解** — 先把拓扑结构画出来，标清各设备位置和作用
2. **区分防火墙和IDS** — 防火墙是主动防御（阻止），IDS是被动检测（告警）
3. **SSL vs IPSec** — 记住"移动用户SSL，站点互联IPSec"
4. **ACL配置** — 注意规则顺序，默认隐含deny
