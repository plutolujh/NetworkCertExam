# 2024年下午案例分析 - 第一题详解
## 网络系统规划设计与优化

---

## 题目回顾

某高校智慧校园网络需要升级改造，当前校园网采用传统的三层架构（核心-汇聚-接入），随着智慧校园业务发展，存在以下问题：
1. 现有网络难以支撑大量物联网终端的接入
2. 网络配置复杂度高，运维效率低
3. 无线网络覆盖不足，存在信号盲区
4. 出口带宽利用率不均衡

---

## 解题思路框架

### 问题1：网络优化措施（4分）

**核心思路**：针对每个现状问题，给出对应的架构层面的优化措施

| 现状问题 | 优化措施 | 关键词 |
|---------|---------|--------|
| 物联网终端接入困难 | 部署SDN控制器 | 集中管控、灵活扩展 |
| 配置复杂度高、运维效率低 | 简化网络配置、标准化VLAN规划 + 部署自动化运维工具 | 自动化、标准化 |
| 无线覆盖不足、存在信号盲区 | 增加无线AP数量 | 无线补盲 |
| 出口带宽利用率不均衡 | 部署多出口负载均衡 | 负载均衡、带宽优化 |

**标准答案示例**：
1. 部署SDN控制器，实现网络集中管控和灵活扩展
2. 简化网络配置，标准化VLAN规划，部署网络自动化运维工具
3. 增加无线AP数量，消除信号盲区
4. 部署多出口负载均衡策略，优化带宽利用率

---

### 问题2：SDN控制器部署（5分）

**SDN控制器部署位置**：
- 方式一：部署在核心层，与核心交换机合设
- 方式二：独立集中部署（推荐），在数据中心或中心机房独立部署

**引入SDN后的改进作用（至少4点）**：
1. **网络可编程**：通过北向接口开放API，实现网络配置自动化
2. **流量调度灵活**：全局视角进行流量工程，优化带宽利用
3. **故障快速响应**：自动化故障检测和恢复
4. **统一管控**：集中管理整个网络，简化运维
5. **灵活扩展**：支持新业务快速部署，无需修改硬件

**答案示例**：
SDN控制器建议部署在核心层或独立部署。其改进作用包括：
1. 实现网络可编程，配置自动化，降低运维复杂度
2. 提供全局网络视图，实现精细化流量调度和负载均衡
3. 支持自动化故障检测和快速恢复，提高网络可靠性
4. 统一管控网络设备，简化配置管理
5. 支持新业务快速部署，提升网络灵活性

---

### 问题3：无线网络部署因素（4分）

**关键考虑因素**：
1. **覆盖范围和信号强度**：根据图书馆面积和布局合理规划AP数量和位置
2. **接入容量和并发用户数**：高密场景需要支持大量并发用户，考虑AP吞吐量和密度
3. **信道规划和干扰规避**：合理规划信道，避免同频干扰，考虑与现有无线网络的干扰
4. **终端兼容性**：支持不同类型终端接入（笔记本、手机、物联网设备）
5. **安全策略**：Portal认证或802.1X认证
6. **供电方式**：PoE供电部署

**答案示例**：
部署高密放装无线网络需要考虑：
1. 覆盖范围和信号强度，确保无盲区
2. 接入容量和并发用户数，满足高密场景需求
3. 信道规划和干扰规避，避免同频干扰
4. 终端兼容性和安全认证策略

---

### 问题4：出口负载均衡策略（2分）

**策略设计**：基于目的地址的策略路由

**实现思路**：
1. 配置教育网地址段列表（ academic.edu.cn 等学术资源域名/IP段）
2. 内网用户访问学术资源时，匹配教育网路由策略，走教育网出口（100Mbps）
3. 其他流量默认走运营商出口（200Mbps）
4. 可以使用PBR（策略路由）或DNS分流实现

**答案示例**：
设计基于目的地址的策略路由（PBR）：
1. 定义教育网地址段列表（如学术资源IP段）
2. 配置策略路由：访问学术资源时下一跳指向教育网出口
3. 其他流量默认走运营商出口
4. 可结合DNS分流，教育网域名解析走教育网DNS

## 答题技巧总结

1. **看清题眼**：题目说"不改变现有三层架构"，所以不能回答改成扁平化大二层等
2. **对应回答**：每个优化措施要明确对应解决哪个问题
3. **专业术语**：使用"部署"、"策略路由"、"SDN控制器"等专业词汇
4. **分点作答**：标清序号，字迹清晰
5. **时间控制**：每题2-3分钟，不要过于展开
